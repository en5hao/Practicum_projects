# Часть 2. Моделирование профиля API

##  Определение цифровых возможностей

У меня активный и любопытный питомец, поэтому я хочу быстро оформить страховку, чтобы быть уверенным в получении помощи в случае болезни.

### Действия и их шаги

| Цифровая возможность | Действие | Шаг действия | Участник | Описание |
|----------------------|----------|---------------|----------|-----------|
| Оформить заказ | Просмотр данных о страховке | Просмотреть данные о страховке | Пользователь | Получение пользователем полной информации о доступных страховых пакетах |
| Оформить заказ | Добавить страховку в корзину | Положить страховку в корзину | Пользователь | Страховка добавляется в корзину как обычный товар (реализовано в сервисе Shopping Cart) |
| Оформить заказ | Оплатить страховку | Оплатить страховку | Пользователь | Оплата через интернет-эквайринг (реализовано в сервисе Payment) |

---

##  Границы API (группировка действий)

| Цифровая возможность | Действие | Шаг действия | Участник | Описание |
|----------------------|----------|---------------|----------|-----------|
| Оформить заказ | Просмотреть данные о страховках | Запросить список страховок | Мобильное приложение | Получение полных данных для выбора страховки: списка всех страховок, сгруппированных по виду животного |

> **Примечание:** Шаги «Добавить в корзину» и «Оплатить» вынесены в отдельные сервисы (`Shopping Cart`, `Payment`) и не входят в границы данного API.

---

##  Профиль API

**Название:** `Insurance API`  
**Краткое описание:** Предоставление информации о страховке для животных  
**Вид API:** Внутренний API

| Название операции | Описание операции | Участники | Ресурсы | Детали операции |
|------------------|-------------------|-----------|---------|------------------|
| `предоставлятьДанныеДляВыбораСтраховки()` | Возвращает все необходимые данные для раздела страховки: описание сервиса и список доступных страховых пакетов с параметрами. | Мобильное приложение | Каталог страховок | **Параметры:** - <br> **Возвращает ответ:** <br> - `описаниеСервиса` <br> - `данныеСтраховок`: <br>   - `типПитомца` <br>   - `описаниеСтраховки` <br>   - `цена` <br>   - `валюта` <br>   - `периодСтрахования` <br> **Безопасная / Синхронная** |
